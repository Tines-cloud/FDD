## Instructions

Specific checks - verify each of these explicitly:
1. Cardinality (min/max): check BOTH min and max for every element.
2. Binding strength AND valueSet URL: a valueSet URL change is drift even if strength is unchanged.
3. Fixed values (fixed[x]): any addition, removal, or change is ERROR-level drift.
4. Pattern values (pattern[x]): any addition, removal, or change is WARNING-level drift.
5. Default values (defaultValue[x]): any addition, removal, or change is WARNING-level drift.
6. meaningWhenMissing: any change is INFO-level drift.
7. Named extension slices: check every `path:sliceName` element in both profiles.
8. Constraint expressions: for constraints with the same key in both profiles, check if the
   expression or severity changed - report as STRUCTURAL drift if so.
9. Type code changes: report if the type code list changes for any element.
10. Reference target profiles: report if targetProfile URLs change on Reference-type elements.

Extension detection - check these specifically:
- Look for elements with paths like `Patient.extension:X` or `Patient.modifierExtension:X` in each profile.
- Any such named slice present in source but not in target -> EXTENSION type, sourcePath="Patient.extension:X", targetPath="".
- Any such named slice present in target but not in source -> EXTENSION type, sourcePath="", targetPath="Patient.extension:X".

Granularity - avoid redundant sub-element expansion:
- When an element (e.g., `Patient.identifier`) is being added to or removed from a profile, report
  ONE item at the parent element level only. Do NOT emit separate items for every child sub-element
  (e.g., `Patient.identifier.id`, `Patient.identifier.use`, etc.) unless those children have
  independent specific differences (cardinality, terminology, or extension changes).

Severity assignment - follow the rules in the system prompt precisely:
  - ERROR   : min cardinality increased, binding tightened to "required", fixed value added/changed, isModifier changed.
  - WARNING : max cardinality tightened, extension added/removed, binding relaxed, constraint expression changed.
  - INFO    : mustSupport/isSummary changed, cardinality relaxed, defaultValue/meaningWhenMissing changed.
